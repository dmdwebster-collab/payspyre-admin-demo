## Quick Start

```bash
# 1. Install Claude Code (if not already installed)
npm install -g @anthropic-ai/claude-code

# 2. Navigate to this repo
cd payspyre-admin-demo

# 3. Launch Claude Code
```

## Session Workflow

### Starting a Session

1. **Open `CONTRACT.md`** before asking Claude to write any code:
   ```bash
   cp CONTRACT.template.md CONTRACT.md
   # Fill in all sections: Goal, Done-When, Constraints, Files In Scope, Interfaces
   ```

2. **Tell Claude** at the start of your prompt:
   > "Read CLAUDE.md, SURFACE.md, and CONTRACT.md first, then enter plan mode."

3. **Review the plan** — confirm it only touches files in this surface (not other PaySpyre surfaces).

4. **Approve execution.**

5. **Run checks** before committing:
   ```bash
   npm run lint && npm run typecheck && npm run test
   ```

6. **Archive CONTRACT.md** after the session:
   ```bash
   mv CONTRACT.md .claude/contracts/CONTRACT_$(date +%Y%m%d_%H%M).md
   ```

## PaySpyre Multi-Surface Awareness

This repo is one of four PaySpyre surfaces:
- **Admin** (admin-demo) — internal ops + practice admins
- **Consumer** (consumer-demo) — patient payment flows
- **Patient Portal** (patient-portal-demo) — patient account + history
- **Vendor Portal** (vendor-portal-demo) — partners + integrators

**Rule:** Claude Code sessions in this repo must ONLY modify files in this repo. Cross-surface changes require a separate session in the correct repo.

When you need to make changes across surfaces, spin up separate Claude Code sessions per repo. Use Claude.ai (Projects) to coordinate architecture decisions across surfaces.

## Key Conventions

| Convention | Rule |
|-----------|------|
| Styles | Tailwind only — no inline styles |
| Validation | Zod on every input (client + server) |
| Components | Server components by default |
| Data scope | Every query scoped by practice_id or patient_id |
| PHI | Never log patient identifiers |
| Secrets | Never in code — process.env server-side only |
| Planning | Always plan mode for >2 file changes |
| Subagents | Propose breakdown for >3 file edits |
